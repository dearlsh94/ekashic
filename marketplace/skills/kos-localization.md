# KOS 다국어 생성

KOS 프로젝트의 lokalise 기반 다국어 시스템에서 새로운 문자열 키를 추가하고 한국어, 일본어 번역을 자동 생성합니다.

## 사용법

```
"키이름" "한국어 원문"
```

예시:
- `"login.welcome_message" "KOS에 오신 것을 환영합니다"`
- `"error.network_failed" "네트워크 연결에 실패했습니다"`
- `"button.save_changes" "변경사항 저장"`

## 기능

- **자동 파일 탐지**: `src/locales/` 디렉토리의 JSON 파일들 자동 확인
- **키 중복 검사**: 기존 키 존재 여부 확인 및 덮어쓰기 옵션 제공
- **자동 번역**: 한국어 원문을 자연스러운 일본어로 변환 (KOS 헬스케어 맥락)
- **영어 키 생성**: 영어 파일에는 키만 추가하고 빈 문자열로 설정
- **JSON 형식 유지**: 기존 파일의 들여쓰기와 정렬 방식 보존

## 구현

이 스킬은 다음 단계를 수행합니다:

### 1. 환경 확인
```bash
# src/locales/ 디렉토리 존재 확인
ls src/locales/

# 언어별 JSON 파일 확인
ls src/locales/ko.json src/locales/ja.json src/locales/en.json
```

### 2. 키 검증
- 기존 JSON 파일에서 중복 키 검사
- 중복 발견 시 사용자에게 덮어쓰기 여부 확인
- 점 구분 키 네이밍 컨벤션 검증 (예: `login.welcome`)

### 3. 번역 생성
**한국어 (ko.json)**
- 제공된 원문을 그대로 사용

**일본어 (ja.json)**
- KOS 헬스케어 서비스 맥락을 고려한 자동 번역
- 정중한 존댓말(敬語) 사용
- 의료/건강 관련 전문 용어 적용

**영어 (en.json)**
- 키만 추가하고 값은 빈 문자열 (`""`)
- lokalise에서 별도 번역 관리 가능

### 4. JSON 파일 업데이트
```json
// ko.json 예시
{
  "login": {
    "welcome_message": "KOS에 오신 것을 환영합니다"
  }
}

// ja.json 예시
{
  "login": {
    "welcome_message": "KOSへようこそ"
  }
}

// en.json 예시
{
  "login": {
    "welcome_message": ""
  }
}
```

## 번역 품질

**일본어 번역 기준**:
- **헬스케어 맥락**: 의료진, 환자, 병원 관련 용어 적절히 사용
- **정중함**: 비즈니스 환경에 적합한 존댓말과 공손한 표현
- **명확성**: 오해 없는 명확하고 간결한 문장 구조
- **일관성**: KOS 서비스 특성에 맞는 톤앤매너 유지

## 오류 처리

- **파일 없음**: `src/locales/` 디렉토리나 JSON 파일이 없는 경우 안내
- **JSON 오류**: 잘못된 JSON 형식 감지 시 파싱 오류 보고
- **권한 문제**: 파일 쓰기 권한이 없는 경우 안내
- **네트워크 오류**: 번역 서비스 연결 실패 시 대체 방안 제시

## 사용 시나리오

### 1. 새 기능 개발
```
"dashboard.patient_count" "총 환자 수"
```
→ 대시보드에 새로운 통계 표시 라벨 추가

### 2. 에러 메시지
```
"error.appointment_conflict" "예약 시간이 중복됩니다"
```
→ 예약 시스템 오류 메시지 다국어 지원

### 3. 사용자 안내
```
"guide.blood_pressure_measurement" "혈압을 측정해주세요"
```
→ 건강 측정 가이드 메시지

## 주의사항

- **lokalise 동기화**: 생성된 번역을 lokalise와 동기화하기 전에 검토 권장
- **의료 용어**: 의료/법률 관련 전문 용어는 전문가 검토 필요
- **키 네이밍**: 점(.) 구분 방식을 사용하여 계층 구조 명확히 표현
- **번역 검증**: 자동 생성된 일본어 번역은 네이티브 검토 권장

ARGUMENTS: $1=키이름, $2=한국어원문

## 실행

주어진 키와 한국어 원문으로 다국어 파일을 생성하겠습니다.

### 단계 1: 환경 확인
```bash
# src/locales 디렉토리 존재 확인
if [ ! -d "src/locales" ]; then
    echo "❌ src/locales 디렉토리를 찾을 수 없습니다."
    echo "📁 KOS 프로젝트 루트에서 실행해주세요."
    exit 1
fi

# JSON 파일들 존재 확인
for lang in ko ja en; do
    if [ ! -f "src/locales/${lang}.json" ]; then
        echo "⚠️  src/locales/${lang}.json 파일이 없습니다. 새로 생성합니다."
    fi
done
```

### 단계 2: 키 중복 확인
기존 JSON 파일들에서 동일한 키가 있는지 확인하고, 있다면 덮어쓸지 사용자에게 확인합니다.

### 단계 3: 번역 생성
- **한국어**: 제공된 원문 사용
- **일본어**: KOS 헬스케어 맥락을 고려한 자연스러운 번역 생성
- **영어**: 키만 추가, 빈 문자열 설정

### 단계 4: JSON 업데이트
각 언어별 JSON 파일을 업데이트하고 형식을 정리합니다.

### 단계 5: 결과 확인
생성된 키와 번역 내용을 표시하고, lokalise 동기화 방법을 안내합니다.

**완료 후 안내사항**:
- 생성된 번역 내용 검토
- lokalise 동기화 실행
- KOS 애플리케이션에서 키 사용 확인
