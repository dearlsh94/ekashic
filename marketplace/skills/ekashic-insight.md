# E-Kashic Insight

일상에서 얻은 인사이트를 월별 테이블에 기록합니다.

## 저장 규칙
- **경로**: `~/.ekashic/insights/YYYY-MM.md` (현재 월 기준 자동 선택)
- **형식**: 마크다운 테이블
- **타임스탬프**: 기록 시점의 현재 날짜/시간 자동 생성 (YYYY-MM-DD HH:MM 형식, KST/UTC+9 기준)
- **월별 관리**: 월이 바뀌면 새로운 YYYY-MM.md 파일 자동 생성

## 테이블 구조

파일이 없으면 헤더 생성 후 행 추가:

| Timestamp | Category | Insight | Context | Impact |
| :--- | :--- | :--- | :--- | :--- |
| 2026-03-05 15:45 | Technical | 인사이트 내용 | 맥락 | 4 |

**참고**: Timestamp는 기록 시점의 실제 현재 시간(KST/UTC+9)으로 자동 생성됩니다.

## 카테고리

### 업무 영역
- Technical: 기술적 발견, 개발 노하우
- Product: 제품 기획, UX 인사이트
- Design: 디자인 원칙, 비주얼 패턴
- Marketing: 마케팅 전략, 고객 심리
- Sales: 영업 기법, 고객 니즈

### 성장 영역
- Strategic: 전략적 판단, 의사결정
- Leadership: 리더십, 팀 관리
- Process: 프로세스 개선, 효율화
- Communication: 소통, 협업 방식
- Learning: 학습 방법, 자기계발

### 개인 영역
- Life: 삶의 지혜, 개인적 깨달음
- Finance: 재무, 투자 인사이트
- Health: 건강, 웰빙
- Relationship: 인간관계

## Proactive 동작

대화 중 다음과 같은 상황이 감지되면 **"이거 인사이트로 기록할까요?"**라고 먼저 제안:

- 사용자가 "깨달았다", "알게 됐다", "발견했다" 등의 표현 사용
- 문제 해결 후 교훈이 될 만한 내용이 나왔을 때
- "이건 기억해둬야겠다", "나중에 써먹어야지" 같은 의도 표현
- 새로운 패턴, 원칙, 노하우가 도출되었을 때

## 실행

1. **사용자의 인사이트 분석**
   - 인사이트 내용과 맥락 파악
   - 적절한 카테고리 선택 (또는 사용자 지정)

2. **타임스탬프 자동 생성**
   ```bash
   # KST (UTC+9) 기준으로 현재 날짜/시간을 YYYY-MM-DD HH:MM 형식으로 생성
   current_timestamp=$(TZ=Asia/Seoul date '+%Y-%m-%d %H:%M')
   ```

3. **월별 파일 자동 선택**
   ```bash
   # KST (UTC+9) 기준으로 현재 월 파일명 자동 결정
   current_month_file="~/.ekashic/insights/$(TZ=Asia/Seoul date '+%Y-%m').md"
   ```

4. **파일 존재 확인 및 처리**
   - 해당 월 파일이 없으면 헤더 포함하여 새로 생성
   - 있으면 기존 파일에 새 행 추가

5. **Write 도구로 저장**
   - 자동 생성된 타임스탬프와 함께 인사이트 기록
   - 월별 파일에 테이블 형식으로 추가

## 중요 규칙

⚠️ **타임스탬프 정확성**: 반드시 기록 시점의 **실제 현재 시간(KST/UTC+9)**을 사용해야 합니다.

⚠️ **월별 파일 관리**: 각 인사이트는 기록된 월의 파일에만 저장됩니다.
- 3월 인사이트 → `2026-03.md`
- 4월 인사이트 → `2026-04.md` (자동 생성)

⚠️ **파일 생성**: 새로운 월의 첫 인사이트 기록 시 자동으로 헤더와 함께 파일 생성
